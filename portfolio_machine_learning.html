Machine LearningChinese Character Number Recognition

Jan - May 2022

This machine learning project is the final project for my graduate level courses in Machine Learning. The project is based off of a simple convolutional neural network classic, handwritten digit recognition, that many people create as one of their first digit recognition machine learning models. The concept is the same, except that instead of the normal numbers that one would write, I used used the model to recognize chinese characters instead. The reason being is that chinese character uses strokes to write, instead of the regular alphabet that most other languages use. The strokes in different writing styles can make it more difficult for the model to recognize, and it is an idea that I wanted to pursue and see what comes of it.

The following picture shows some sample characters that are used to train the model. Most handwritten digit recognition models uses the handwritten digit dataset from MNIST, but I had curated my own small set of data to train with. I had used the numbers one through nine, each with around 15 images in different font styles to factor in variances in the same character. Each image is labeled with the correct classification of the image.

The mock welcome page includes two buttons, Exit and Log In. If you click on Exit, it would trigger a VBA command to exit out of the excel workbook. If you click on Log In, it would trigger a pop up asking for a username and password to access the sales data. The user name is "admin" and the password is "password1". The username and password is checked against the only username and password combination in the VBA code.

Once the correct username and password combination is entered, you are met with the sales data sheet, which contains information such OrderID, Date, customer Name, City and State, as well as what is bought and the quantity for it. There is a 'Go to Dashboard' button on the top right which will take you to the dashboard page, which will show you all the metrics the dashboard is tracking.

The dashboard shows Total Sales, Total Transactions, and Average Amount for the data in the Sales Data sheet. On the left hand side, there are filters for Year and Gener, which will filter the data on the Sales Data sheet and the dashboard will reflect these changes. You can filter by different years and different geners. Also on this left hand side is the tabs for Dashboard and State. Clicking on these will take you to the Dashboard sheet or the State sheet, which will show you the total sales for each of the States given the filters you have applied.

The state map shows a heatmap distribution based on the filters you applied on the Year you want to see and the Gender you have selected.

The dashboard itself needs lots of improvements, however I think it is a decent application of what I was taught in the class. There were lots of hours spent on trying to figure out what went wrong in the back-end of the program, learning VBA was a little difficult, however it got easier with time.
